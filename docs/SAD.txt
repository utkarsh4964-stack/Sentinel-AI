---

# 1. Purpose

This document defines the complete system architecture of Sentinel AI.

It specifies:

* Components
* Data flow
* Trust boundaries
* Services
* Agent interactions
* Deployment
* Security
* Communication

This document is the architectural source of truth.

---

# 2. Architectural Principles

1. Modular

Every subsystem is independently replaceable.

---

2. Stateless APIs

FastAPI servers remain stateless.

State is stored only in databases.

---

3. Asynchronous Processing

Long running scans never block HTTP requests.

---

4. AI is advisory

LLMs never create findings.

Static analysis tools remain the source of truth.

---

5. Explainability

Every AI response must cite evidence.

---

6. Replaceability

Every external dependency can be replaced without changing business logic.

# 10. Security Model

Temporary clone directory.

Automatically deleted after scan.

Secrets never logged.

Prompt contains only required code snippets.

Every external API request authenticated.

No direct database access from agents.

No AI calls from controllers.

# 26. Architecture Definition of Done

Architecture is complete when

Every component has one responsibility.

Every dependency is replaceable.

Every service has tests.

Every communication path is documented.

Every trust boundary is defined.

Every failure mode has recovery.

Every external integration is abstracted.
