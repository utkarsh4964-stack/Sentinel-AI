# Sentinel AI

# Product Requirements Document (PRD)

## Version 1.0 (MVP)

---

# 1. Product Overview

## Product Name

Sentinel AI

## Product Type

AI-powered Security Review Platform for GitHub Repositories and AI-Generated Applications.

## Product Goal

Enable developers to discover, understand, and fix security vulnerabilities in their software before deployment using a combination of static analysis, repository understanding, and LLM reasoning.

---

# 2. Problem Statement

Modern developers increasingly rely on AI coding tools such as Claude Code, Cursor, GitHub Copilot, and Lovable.

These tools generate software quickly but often introduce:

* Security vulnerabilities
* Exposed secrets
* Broken authentication
* Poor authorization
* Dangerous AI workflows
* Technical debt

Existing security tools generate findings but rarely explain:

* Why the issue exists
* Its business impact
* How to fix it
* Whether it is a false positive

Sentinel solves this gap.

---

# 3. Product Objectives

The MVP must allow a developer to:

✓ Connect a GitHub repository

✓ Scan the repository

✓ Detect security issues

✓ Explain each issue

✓ Prioritize issues

✓ Suggest code fixes

The MVP will NOT automatically modify repositories.

# 7. Functional Requirements

GitHub Repository Import

User enters

Repository URL

System validates repository.

Supports

* Public repositories

Private repositories are out of scope for MVP.

---

Repository Cloning

System clones repository.

Repository stored temporarily.

Deleted after analysis.

---

* Directory structure

* Frameworks
* Dependencies

* Database access

Normalize output.

# 17. Acceptance Criteria

The MVP is complete when a user can:

1. Paste a public GitHub repository URL.

2. Receive a repository summary.

3. Receive a security score.

4. View categorized vulnerabilities.

5. Read AI explanations for every finding.

6. Export a Markdown report.

7. Complete the entire workflow without using the terminal.

# 19. Definition of Success

A developer with no security expertise should be able to scan a repository and understand:

* What security problems exist
* Why they matter
* Which problems should be fixed first
* How to begin fixing them

without reading documentation for Semgrep, Bandit, or other security tools.

If Sentinel achieves this, the MVP has succeeded.
