# Agents Specification Document (Agents.md)

**Version:** 1.0

---

# 1. Purpose

This document defines every AI agent used within Sentinel AI.

Each agent has:

* A single responsibility
* Defined inputs
* Defined outputs
* Strict operational boundaries
* Prompt specification
* Failure handling
* Quality constraints

Agents must never perform tasks outside their responsibility.

The Scan Orchestrator coordinates all agents.

Agents never call each other directly.

# 16. Prompt Engineering Rules

Every prompt must:

* Use system prompts.
* Return JSON only.
* Forbid markdown.
* Forbid speculation.
* Require evidence-based reasoning.
* Explicitly reject unsupported conclusions.

---

# 18. Security Constraints

Agents must never:

* Execute repository code.
* Modify repositories.
* Access secrets outside scan scope.
* Invent vulnerabilities.
* Ignore scanner output.
* Make network requests except approved APIs.

---

# 19. Observability

Every agent logs:

* Agent ID
* Scan ID
* Start Time
* End Time
* Duration
* Tokens Used (LLM agents)
* Success/Failure
* Retry Count

---

# 20. Definition of Done

An agent is complete only if it includes:

* Typed input model
* Typed output model
* System prompt
* Unit tests
* Integration tests
* Logging
* Retry strategy
* Error handling
* Documentation
* JSON schema validation
* No side effects outside its responsibility
