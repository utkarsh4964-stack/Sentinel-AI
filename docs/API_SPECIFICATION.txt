# Sentinel AI

# API Specification Document (API_Spec.md)

**Version:** 1.0 (MVP)

**Protocol:** REST

**Data Format:** JSON

**Authentication:** None (MVP), JWT (v1.1)

**API Version:** `/api/v1`

---

# 1. Purpose

This document defines all external REST APIs exposed by Sentinel AI.

It serves as the contract between:

* Frontend
* Backend
* Future CLI
* Future VS Code Extension
* Third-party integrations

No frontend component may directly access the database.

All communication must occur through these APIs.

---

# 2. API Design Principles

Sentinel follows an Action-Oriented API design.

APIs represent **user actions**, not database tables.

Example:

❌ POST /repositories

❌ POST /repository/import

✅ POST /scan

Because the user wants to **scan a repository**, not create database records.

---

# 3. Response Format

Every successful response follows:

```json
{
    "success": true,
    "message": "Scan started successfully.",
    "data": {},
    "meta": {}
}
```

Every error response follows:

```json
{
    "success": false,
    "message": "Repository not found.",
    "error": {
        "code": "REPOSITORY_NOT_FOUND",
        "details": "The provided GitHub repository does not exist."
    }
}
```

# 7. Scan APIs

These are the primary APIs of the platform.

---

## POST /scan

### Purpose

Start a repository security scan.

### Request

```json
{
    "github_url": "https://github.com/user/project"
}
```

---

### Processing Pipeline

1. Validate URL

2. Clone repository

3. Create snapshot

4. Parse repository

5. Generate embeddings

6. Run security tools

7. AI reasoning

8. Calculate score

9. Generate report

---

### Response

```json
{
    "scan_id": "scan_001",
    "status": "QUEUED"
}
```

---

## GET /scan/{scan_id}

Returns scan status.

---

### Example

```json
{
    "scan_id": "scan_001",
    "status": "PARSING",
    "progress": 46
}
```

---

### Possible Status

QUEUED

CLONING

PARSING

EMBEDDING

SCANNING

REASONING

REPORTING

COMPLETED

FAILED

---

## GET /scan/{scan_id}/summary

---

    "critical": 1,

    "high": 3,

    "medium": 7,

# 19. API Security Rules

* Validate every GitHub URL.
* Reject unsupported repository types.
* Reject repositories exceeding size limits.
* Never expose internal file paths.
* Never expose stack traces.
* Sanitize all user inputs.
* Enforce request validation with Pydantic models.

---

# 20. API Design Constraints

Controllers must remain thin.

Controllers only:

* Validate requests.
* Call services.
* Return responses.

Business logic belongs exclusively in the Service Layer.

Controllers must never:

* Access the database directly.
* Invoke LLMs directly.
* Execute static analysis tools.
* Parse repositories.
