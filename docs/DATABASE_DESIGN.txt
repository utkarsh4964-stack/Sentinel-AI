# Sentinel AI

# Database Design Document (DDD)

Version 1.0

---

# 1. Database Architecture

Sentinel uses a polyglot persistence architecture.

```
                Sentinel

                    │

      ┌─────────────┼─────────────┐

 PostgreSQL      ChromaDB      Redis

 Structured      Embeddings    Cache / Queue

```

Each database has one responsibility.

# 24. Retention Policy

Repository clone

Delete after scan.

Metadata

Permanent.

Reports

Permanent.

Embeddings

Regenerated on rescan.

Logs

90 days.

---

# 25. Database Constraints

Every repository belongs to one user.

Every scan belongs to one repository.

Every finding belongs to one scan.

Every explanation belongs to one finding.

No orphan records.

Cascade deletes disabled.

Soft deletes preferred.
