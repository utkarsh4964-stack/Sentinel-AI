# Sentinel AI

# Development Roadmap (Development_Roadmap.md)

**Version:** 1.0

**Duration:** 6 Weeks

**Goal:** Build a production-ready MVP capable of scanning public GitHub repositories, detecting security issues, explaining findings with AI, and generating professional reports.

---

# 1. Guiding Principles

Every week must produce:

* A working application increment
* Passing automated tests
* Updated documentation
* A deployable build
* GitHub commits with meaningful history

Do not begin the next milestone until the current milestone satisfies its Definition of Done.

---

# 2. Project Timeline

```text
Week 1 → Foundation

Week 2 → Repository Intelligence

Week 3 → Security Analysis

Week 4 → AI Reasoning

Week 5 → Frontend & Reports

Week 6 → Polish, Deployment & Release
```

# 7. Risks & Mitigation

| Risk               | Mitigation                                     |
| ------------------ | ---------------------------------------------- |
| LLM rate limits    | Cache responses and implement retries          |
| Large repositories | Enforce repository size limits for MVP         |
| False positives    | Rely on static analyzers as source of truth    |
| Performance issues | Background workers and asynchronous processing |
| Scope creep        | Freeze MVP features until v1.0 release         |

---

# 8. MVP Exit Criteria

Sentinel MVP is complete when:

* A user can submit a public GitHub repository URL.
* The repository is cloned and analyzed.
* Static analysis tools identify supported vulnerabilities.
* AI explains findings using repository context.
* A security score is calculated.
* Findings are displayed in the frontend.
* Reports can be exported.
* The application is publicly deployed.
* Documentation is complete.
* CI/CD pipeline passes.
* All critical bugs are resolved.
